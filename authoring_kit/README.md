# Octavia Authoring Kit (v0.3)

Пакет знаний для ИИ: **чертёж движка** Octavia — как устроены данные,
зачем так сделано, как граф с ними живёт. Не шпаргалка «запомни Sum и 128».

## Зачем

Открываешь **только эту папку** в Cursor. Агент не видит Python аддона.
Цель — чтобы он **понял модель**, а не выучил ответы на известные вопросы.

## Как получить папку

В Blender: Node Editor → N → вкладка **Octavia VSD** → **ПАПКА ДЛЯ СЕССИИ С ИИ**.
Выбираешь директорию — туда копируется этот kit. Слепки графа («СНЯТЬ СЛЕПОК ГРАФА»)
дублируются сюда в подпапку `graph_snapshots/` (и параллельно в аддон).

## Как тестировать

1. Open Folder → эту папку (не весь аддон).
2. Новая сессия.
3. Задавай **неожиданные** вопросы (не из README и не «перескажи буфер»).
   Примеры духа: «что будет, если…», «как бы ты сделал эффект X», «где сломается…»,
   «почему нельзя Y», «что должно произойти после отпускания клавиши».
4. Смотри **рассуждение**, не совпадение с формулировками из правил.
5. Дыры в понимании → правим чертёж в исходном `authoring_kit` аддона и снова
   жмём кнопку (перезапишет файлы kit).

Хороший знак: агент выводит следствие из модели. Плохой: дословно цитирует
таблицы и «золотые правила» без «почему».

## Что внутри

| Файл | Содержание |
|------|------------|
| `.cursor/rules/octavia-engine.mdc` | Чертёж движка: смысл, поток данных, роли |
| `.cursor/rules/octavia-agent-role.mdc` | Как рассуждать (не цитировать догму) |

| `graph_snapshots/` | Слепки графа (появляются после «СНЯТЬ СЛЕПОК ГРАФА») |

## Версия

`v0.3` — после рабочего тестового графа со смещением и возвратом: в чертёж
добавлены фазы после отпускания, вклад как огибающая, полки макросов в DAW,
призрак таймлайна vs геометрия, слепки как факт сборки. По-прежнему без
рецептов нод и без эталона «скопируй граф».
